## 1. 设计目标

当前朝会、黄门密札、西园精舍虽然已经完成 1100×640 三栏信息架构，但外观仍过于接近：同一类深色不透明 Panel、相似金边、相似标题和按钮。下一步要让玩家一眼分辨“现在进入的是哪种权力空间”。

一句话目标：**弹窗不是同一套工具面板换标题，而是不同宫廷机构的场景化界面。**

- 朝会：帝国公开权力，黑金朱红、肃穆、纵深、压迫感。
- 黄门密札：秘密情报与舆图，纸张、密函、朱砂、墨迹。
- 西园精舍：天子私军与私库，军帐、青铜、铁色、兵符。
- 起驾确认/小提示：黄门传令，小而锋利，像手递奏签。
- 大臣详情：人物档案/奏牍，不是现代属性卡。
- 行动确认：圣裁前最后一层风险提示，突出代价与不可逆感。

---

## 2. 全局原则

### 2.1 统一骨架，不统一皮肤

保留现有 `WindowManager.PushWindow(...)` 的不透明、防穿透、ESC 退栈机制；保留 1100×640 大面板规格。变化集中在：

- 根 Panel 背景材质
- 内栏 Panel 样式
- 标题区装饰
- 按钮样式
- 小标签/风险色
- 局部纹理或低对比背景

### 2.2 不再继续使用 emoji 作为核心视觉

现有标题如 `👑 宣政殿 · 大朝会`、`🏯 西园精舍` 可视为临时占位。后续改为：

- 小型 TextureRect 场景符号，或
- 纯文字标题 + 左右纹样/印章

避免继续扩展 `📜/🏯/🐎` 这类 emoji。

### 2.3 文字可读性优先

所有弹窗都可能承载长文本，不能为了氛围牺牲阅读：

- 正文区域对比度要稳定。
- 背景纹理必须低对比、低饱和。
- 大段 RichTextLabel 区域优先用纸面/暗底纯色块。
- 金色只用于标题、关键资源、边框，不铺满正文。

## 3. 弹窗类型分层

### A 类：核心场景大弹窗

对象：

- 朝会 `CourtPopup`
- 黄门密札 `IntelPopup`
- 西园精舍 `WestGardenPopup`

规格：

- 尺寸：1100×640，居中。
- 结构：顶部标题 + 三栏主体。
- 视觉：强场景差异。
- 关闭：场景化按钮（退朝 / 收起密札 / 合上军簿）。

### B 类：二级详情弹窗

对象：

- 大臣详情 `MinisterOverlayPanel`
- 阅兵/募兵/招安/平叛等局部操作弹窗

规格：

- 尺寸：约 520×360 到 720×460。
- 结构：标题 + 关键数据 + 操作按钮。
- 视觉：跟随来源场景，但更像文书卡片。

### C 类：确认/提示小弹窗

对象：

- 起驾确认
- 行动不可用提示
- 高风险决策确认

规格：

- 尺寸：约 420×220 到 560×280。
- 结构：黄门急奏/内侍传令 + 简短说明 + 双按钮。
- 视觉：奏签、短札、传令牌。

## 4. 核心场景样式方向

## 4.1 朝会样式：黑金朱红 · 宣政殿压迫感

### 气质关键词

- 高处御座
- 群臣阴影
- 黑金宫殿纵深
- 朱红梁柱
- 肃穆、公开、压迫、等级森严

### 色彩 token

- 背景：`#120807` 近黑朱红
- 主面板：`#1A0D08`
- 内栏：`#23110A`
- 边框金：`#B4862A`
- 暗红强调：`#6E1E16`
- 正文浅金：`#E4C47A`
- 次级灰金：`#A8915D`

### 组件表现

- 根面板：暗红黑底 + 较粗金边，顶部可有低对比宫殿纵深纹理。
- 三栏：像殿柱分隔，边框偏硬，少圆角。
- 标题：`宣政殿 · 大朝会`，字体更大，左右可放短横金线。
- 左栏百官：人物按钮像“朝班名牌”，hover 金边。
- 中栏议题：议题卡片像奏章被呈上，重点议题可带朱砂左边条。
- 右栏裁断：按钮要像“圣裁牌”，危险决策用暗红底。

### 适合的资产

- `court_throne`：御座 + 群臣阴影剪影，作为标题侧图或低透明背景。
- 低对比宫殿纵深纹理，不放具体人脸。

---

## 4.2 黄门密札样式：密函纸面 · 舆图情报台

### 气质关键词

- 黑漆密函
- 米黄纸张
- 朱砂封印
- 舆图、密录、墨迹
- 秘密、冷静、可分析

### 色彩 token

- 外框黑漆：`#0E0B08`
- 纸面：`#D8C49A`
- 深纸面：`#BCA676`
- 墨色：`#24180F`
- 朱砂：`#9E2F1D`
- 警戒黄：`#B88420`
- 安定青绿：`#3C7A5A`

### 组件表现

- 根面板：黑漆盒外壳，内部像铺开密札/舆图。
- 顶部天下总览：像密报抬头，浅纸底黑字，不再用纯深色 RichText。
- 左栏州郡列表：像一叠密札签条，叛乱州用朱砂印，危急用黄签。
- 中栏详情：纸面阅读区，适合长文本。
- 右栏处置：仍可暗底，因为是“密令操作区”；按钮像朱批/签发。

### 适合的资产

- `intel_secret_chamber` 或黑漆密函图标。
- 低对比舆图纸纹，可用于中栏背景。

---

## 4.3 西园精舍样式：军帐铁铜 · 私军密署

### 气质关键词

- 军帐
- 青铜虎符
- 铁甲与兵簿
- 私库账册
- 烛火、军令、危险的亲军权力

### 色彩 token

- 军帐暗褐：`#14100B`
- 铁色：`#2A2B2A`
- 青铜：`#8A6A32`
- 暗绿铜锈：`#334138`
- 皮革棕：`#4A2B17`
- 火光金：`#D19A3A`
- 危险红：`#8A251C`

### 组件表现

- 根面板：暗军帐底 + 青铜/铁边框，比朝会更粗粝。
- 左栏军势：像军簿/兵牌，人数士气忠诚可以用条形进度感。
- 中栏校场与私库：叙事区可用暗皮革/纸账册混合。
- 右栏军务处置：按钮像军令牌/虎符签发；募兵、发饷突出资源代价。
- 关闭按钮：`合上军簿` 保持，比普通“关闭”更场景化。

### 适合的资产

- `west_garden_tiger_tally`：标题图标或按钮主符号。
- 军帐/兵符低对比背景。

## 5. 二级弹窗样式

## 5.1 大臣详情：奏牍档案卡

当前大臣详情容易像属性窗口。建议改成“人物奏牍”：

- 背景：深纸面或黑漆边框 + 纸质内芯。
- 标题：姓名 + 官职，如 `何进 · 大将军`。
- 五维/好感/权势：不要像现代表格；用短句 + 标签：
  - `权势：炽盛`
  - `贪腐：可疑`
  - `态度：请战`
- 操作：`召见` / `抄家` / `退下` 这种短命令。

来源场景差异：

- 朝会打开：偏黑金奏牍。
- 西园打开：偏军簿。
- 情报打开：偏密档。

第一轮可先统一为“奏牍档案卡”，不做来源差异。

## 5.2 行动参数弹窗：圣裁前置卡

对象：发饷、募兵、平叛、招安。

原则：

- 上半区显示“将发生什么”。
- 中间是参数控件（SpinBox/CheckBox/OptionButton）。
- 下半区固定显示代价预览。
- 最底部是 `颁诏执行` / `暂缓`。

视觉：

- 平叛/募兵：西园军令牌风格。
- 招安/任免：密札朱批风格。
- 朝会裁断：圣裁朱批风格。

## 6. 小提示与确认弹窗

## 6.1 起驾确认：黄门急奏短札

示例：

```text
【黄门急奏】
陛下尚未驾临宣政殿，百官不可无端入朝。

[起驾宣政殿] [暂缓]
```

样式：

- 小尺寸居中。
- 外壳黑漆，内部短札纸面。
- 标题用朱砂色。
- 主按钮金边；取消按钮低对比。

## 6.2 高风险确认：朱批警示

用于未来高风险动作，如大规模出兵、国库不足仍强行筹饷、处死重臣等。

样式：

- 暗红边框。
- 代价用红色/金色突出。
- 按钮文案避免现代“确认/取消”，用 `朱批照准` / `收回成命`。

## 9. 防回退实现注意事项

`ApplyOpaquePanelTheme(this)` 会递归扫描场景树，给未显式设置样式的 `Panel` 补不透明兜底背景。已经调用 `CreatePopupPanelStyle(...)` / `CreatePopupInnerPanelStyle(...)` / 其他自定义 `AddThemeStyleboxOverride("panel", ...)` 的弹窗和卡片，不能再被兜底灰黑金边覆盖。

实现约束：

- `ApplyOpaquePanelTheme` 只能在 `!panel.HasThemeStyleboxOverride("panel")` 时添加 `CreateOpaquePanelStyle(...)`。
- 新增核心弹窗必须先设置专属 `PopupSkin`，再入栈。
- 不要在 `WindowManager.PushWindow(...)` 或全局主题函数中无条件覆盖 `panel` stylebox。
- 验收时除 build/test 外，要实际 Godot headless 检查日志无错误；视觉验收则优先截图确认不同弹窗颜色/材质没有退化成同一种灰黑面板。
