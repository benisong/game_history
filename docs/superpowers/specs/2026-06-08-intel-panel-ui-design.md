## 1. 设计结论

黄门密札不应只是“州郡列表 + 右侧操作按钮”，而应成为玩家查看天下硬数据、识别危机、处理地方事务的核心情报台。

一句话目标：**主界面像御案，朝会像朝堂，黄门密札像情报司。**

它承担三件事：

1. **天下硬数据总览**：皇权、国库、私库、民心、健康、西园军势、叛乱数量等集中展示。
2. **州郡风险研判**：每个州郡给出风险等级、叛乱状态、太守状态、民心/防务摘要。
3. **治理行动闭环**：任免太守、军事平叛、遣使招安仍在这里执行，但要分组清晰、先看后做。

## 2. 当前问题审计

当前 `MainScene.IntelPanel.cs` 已具备基础功能：

- 顶部显示天下民心 + 西园军势。
- 左侧 ItemList 显示州郡列表。
- 右侧显示州郡详情。
- 可任命 / 召回太守。
- 叛乱时可军事平叛，支持出兵规模、军费、胜率预览。
- 叛乱时可遣使招安，支持四策组合和赈灾金额。

但玩家体验仍有明显问题：

### P0 / P1

暂无明显会导致错误行为的问题。

### 影响游戏性的 P2

1. **面板尺寸太小**
   - 当前 720×450，对“天下总览 + 州郡列表 + 详情 + 行动区”明显拥挤。
   - 朝会已升级到 1100×640，情报面板也应提升到近似规格。

2. **硬数据总览不足**
   - 当前只显示天下民心、西园军人数/士气/忠诚。
   - 玩家还需要同时看到皇权、国库、私库、健康、当前所在地、叛乱州数。

3. **州郡列表信息密度低但不可读**
   - 目前每行是：`州名 (太守) 状态`。
   - 缺少风险等级、民心、防务、距京、叛军规模提示。

4. **右侧信息没有层次**
   - 州郡详情、任免、军事平叛、招安都堆在一个 VBox 中。
   - 玩家难以区分“情报研判”和“马上执行的操作”。

5. **空状态过于工具化**
   - “请选择一个郡县...”像开发测试提示。
   - 应改成黄门密札风格，同时提示玩家先看哪里。

6. **叛乱行动预览很好，但非叛乱州缺少治理建议**
   - 安定州只显示任命/召回，缺少“风险研判”：低民心、低防务、无太守等预警。

7. **没有情报记录区**
   - `GameState.IntelReports` 已存在，但界面未展示。
   - 后续旬更情报、百官密录、天下异动应在密札中可见。

## 3. 第一轮实现范围

第一轮目标：**不新增后端规则，只重构 UI 信息架构与展示。**

### 做

- 面板改为 1100×640 三栏布局。
- 顶部增加“天下总览”横条。
- 左栏改为“州郡密札”列表，显示风险标签。
- 中栏显示“州郡详情 / 风险研判 / 情报记录”。
- 右栏显示“可行处置”，把任免、平叛、招安分组展示。
- 保留现有后端调用：
  - `AssignGovernor(...)`
  - `RecallGovernor(...)`
  - `SuppressRebellion(provinceId, generalId, troops)`
  - `PacifyRebellion(provinceId, envoyId, strategies, reliefGold)`
- 保留军事平叛的出兵规模 SpinBox 与胜率动态预览。
- 保留招安四策 CheckBox 与赈灾金额 SpinBox。

### 不做

- 不新增州郡治理规则。
- 不新增税收、征兵、修城等后端动作。
- 不接真实 AI 情报系统。
- 不做地图绘制或复杂地理可视化。
- 不改变平叛/招安公式。

## 5. 顶部：天下总览

显示一行或两行硬数据：

```text
黄门密札 · 天下情报台
光和7年 · 184年4月上旬
皇权 25/100｜国库 8000万｜私库 1200万｜民心 28/100｜龙体 35/100｜所在地 宣政殿
叛乱州 1/6｜无太守州 2/6｜西园新军 8000人｜士气 55｜忠诚 50
```

颜色建议：

- 红色：危险（民心 < 25、健康 < 30、叛乱中）
- 黄色：警戒（民心 < 40、防务 < 40、无太守）
- 绿色：安定
- 金色：标题与关键天子资源

---

## 6. 左栏：州郡密札

每个州郡用两行信息：

```text
⚡ 冀州  黄巾叛乱
民心18｜守军2000｜太守无｜距京3
```

风险标签：

- `⚡ 叛乱`：`IsRebelling == true`
- `⚠ 危急`：民心 < 25 或无太守且民心 < 35
- `△ 不稳`：民心 < 40 或防务 < 40
- `○ 安定`：其余

排序：

1. 叛乱州优先
2. 危急州
3. 不稳州
4. 安定州
5. 同级内按距京从近到远

这样玩家打开密札后先看到最该处理的问题，而不是单纯按距离排序。

## 7. 中栏：详情与风险研判

选择州郡后，中栏显示：

1. 州郡基本信息
2. 风险研判
3. 近日密录 / 相关情报

示例：

```text
【冀州】距京：3
当前局势：⚡ 黄巾叛乱，已持续 2 个月
地方太守：暂无
地方民心：18 / 100
郡中守军：2000 人
郡县财富：3000 万
防务等级：30 / 100
邻接州郡：司隶、兖州、并州

【黄门研判】
- 民心低于 25，黄巾响应风险极高。
- 当前无太守，地方恢复能力不足。
- 守军薄弱，若久拖可能向邻郡蔓延。

【近日密录】
- 暂无专属密录。
```

`GameState.IntelReports` 第一轮可显示最近 3 条全局情报；如果未来情报带 provinceId，再筛选专属情报。

## 8. 右栏：可行处置

右栏只放按钮和行动预览，不再混杂详情文字。

### 8.1 太守任免

- 有太守：显示召回按钮。
- 无太守：显示最多 4 个候选人。
- 候选人按钮显示：姓名、政治、野心、派系。

```text
【太守任免】
[任曹操 政85 野75 清流派]
[任蹇硕 政20 野40 阉党派]
```

### 8.2 军事平叛（仅叛乱州）

保留当前优秀设计：

- 出兵规模 SpinBox
- 军费预览
- 将领战力、距京惩罚、兵力修正、预计胜率
- 出征按钮

但视觉上放在独立分组：

```text
【军事平叛】
出兵规模：3000 人
军费：300 万
[命曹操出征  胜率72%]
[命蹇硕出征  胜率44%]
```

### 8.3 遣使招安（仅叛乱州）

保留当前四策：

- 离间计
- 说服
- 赈灾放粮
- 惩治官员

并增加一行预览文案：

```text
预计策略：说服 + 赈灾 500万
```

第一轮可以只显示静态预览，不必复刻后端成功率公式。

## 9. 空状态

打开密札但未选州时，中栏显示：

```text
【黄门密札】
天下州郡、民心、守军、太守与叛乱皆录于此。

请在左侧选择一州，查看地方风险与可行处置。

黄门提醒：
- 叛乱州优先处置。
- 无太守且民心低的州，易生民变。
- 西园军虽为天子亲军，但出兵会消耗国库。
```

右栏显示：

```text
【可行处置】
先选择州郡。
```

## 10. 文件拆分建议

当前 `MainScene.IntelPanel.cs` 约 314 行，第一轮重构后会接近或超过 500 行。建议拆为：

- `MainScene.IntelPanel.cs`
  - 初始化布局
  - 打开/关闭
  - 总体渲染编排
- `MainScene.IntelProvince.cs`
  - 州郡列表、详情、风险研判文本
- `MainScene.IntelActions.cs`
  - 太守任免、军事平叛、遣使招安 UI 与回调

第一轮也可先不拆，但为了延续朝会拆分后的风格，建议直接拆。

## 11. 验收标准

- 打开“📜 情报”后，看到 1100×640 左右的大面板。
- 顶部能看到皇权、国库、私库、民心、健康、当前位置、叛乱数量、西园军势。
- 左侧州郡列表按风险排序，叛乱/危急一眼可见。
- 选择州郡后，中栏有清晰详情与风险研判。
- 右栏只显示可执行处置，任免/平叛/招安分组明确。
- 原有任命太守、召回太守、军事平叛、遣使招安功能仍可用。
- 构建通过：`cd donghan/Frontend && dotnet build DonghanFrontend.csproj -v minimal`
- 后端测试通过：`cd donghan/Backend && dotnet test DonghanEngine.Tests/DonghanEngine.Tests.csproj`
