# 武侠传承录 (Wuxia Inheritance) —— 世代传承文字挂机游戏设计规格书

## 1. 游戏概述
《武侠传承录》是一款融合了 **MUD（文字江湖）网格移动探索** 与 **放置挂机（Idle）** 机制的国风武侠角色扮演游戏。
游戏的核心灵魂在于**“代代相传，薪火不熄”**。玩家不仅可以通过挂机、采集、打工来提升实力、积累财富，还能手操角色在水墨网格大地图上探索江湖、历练除恶、结识红颜。最核心的特色是当角色达到一定境界后可领养孩子，将衣钵传给下一代。上一代角色会变成游戏世界中的实体 NPC 逐渐老死，而下一代孩子不仅能继承家传绝学，还能随时找上一代父辈 NPC “传功”以快速崛起。

## 2. 核心架构与技术栈 (方案 B)
* **后端 (Server)**: Node.js (Express)
  * 职责：负责存档数据安全、账号注册/登录、离线挂机时间比对与收益结算、数值安全性校验。
  * 主要 API 路由：
    * `/api/auth`: 注册、登录、存档重置。
    * `/api/game/state`: 读取与保存当前玩家的所有状态（属性、背包、功法、红颜、大地图坐标、传承历史）。
    * `/api/game/offline`: 比对离线时间，结算离线修为/银两/资源。
    * `/api/map`: 返回各张大地图的静态网格配置（包含障碍、NPC 初始位置、资源点等）。
* **前端 (Client)**: HTML5 + CSS3 (国风水墨主题) + Vanilla JS (ES Modules) + HTML5 Canvas
  * 职责：渲染多侧重 UI（创角、大地图与战斗一体化、挂机控制、红颜、装备与锻造、轮回）。
  * 地图渲染：Canvas Tilemap（文字/水墨图标，带 Camera 视口边缘精确裁剪、迷雾遮罩绘制、室内外动态缩放倍率）。
  * 战斗渲染：Canvas 国风火柴人半即时动作绘制 + 大地图正下方文字战斗日志（Combat Log）。

## 3. 属性、年龄与寿命系统
### 3.1 基础属性 (新五维)
角色开局拥有 **20点天命点数**，可自由分配。基础五维属性如下：
1. **根骨 (Bone)**: 
   * **唯一**决定主角的生命值（HP）上限与基础最大寿命。
   * 基础最大寿命 = `60 + 根骨 * 2` 岁。
   * HP 上限 = `100 + 根骨 * 20`。
2. **悟性 (Savvy)**: 
   * **唯一**掌控功法修炼的天资。
   * 悟性越高，升级/参悟任何武学所需的**修为消耗越低**，且需要的**挂机参悟时间越短**。
   * 减免公式：`实际消耗/时间 = 原始消耗/时间 * (1 - 悟性 / 40)`（减免最大上限为 75%）。
3. **力量 (Strength)**:
   * **唯一**决定战斗中的基础伤害和招式物理威力。
   * 物理攻击力 = `10 + 力量 * 5`。
4. **敏捷 (Agility)**:
   * **唯一**决定出手速度（战斗进度条积累）和闪避率。
   * 闪避率 = `敏捷 * 1%`。
5. **家境 (Wealth)**: 
   * 决定主角初始自带的银两数量。

### 3.2 年龄与死亡机制
* **初始年龄**: 15 岁。
* **时间流逝**: 游戏内每 10 分钟（可配置）真实时间流逝一岁，或者在大地图移动/挂机按比例折算年龄。
* **寿终正寝**: 一旦主角达到由根骨决定的最大寿命上限，将强制死亡，必须进入衣钵轮回。
* **战败殒命**: 若大地图战斗或死斗中生命值归零且无保命丹药，亦会面临道殒，被迫提前传位给孩子。

## 4. 挂机与资源循环系统
挂机系统分为**在线/离线挂机**，挂机时只能在以下三种行为中**三选一**：
1. **闭关修炼 (Cultivate)**:
   * **消耗**: 每秒消耗一定量的 **食物**（若食物不足，则按 1 食物 = 1 银两自动扣除对应银两；若连银两也不够，则挂机效率降低至 20%）。
   * **产出**: 持续积累 **修为/真气**（用于提升人物境界、突破武学层数）。
2. **江湖采集 (Gather)**:
   * **消耗**: 无。
   * **产出**: 持续积累 **制造资源**（如：铁矿石、玄铁、百草，用于锻造或炼丹）。
3. **镖局打工 (Work)**:
   * **消耗**: 无。
   * **产出**: 持续积累 **银两**。

### 4.1 历练与修为转换 (主动成长)
* **历练 (Experience)**: 通过大地图行侠仗义、击败恶霸、完成固定剧情或日常任务获得。
* **丹药转换**: 历练本身不能直接提升境界或升级功法。玩家需要用银两在各大城镇药铺购买“丹药”（如：聚气散、洗髓丹）。服下丹药后，会将积累的“历练”按一定比率转化为“修为”。

## 5. 50x80 大迷宫、室内外倍率缩放与精确边界摄像机
### 5.1 50x80 水墨迷宫生成
* 户外大地图拓展为庞大的 `50 x 80` 网格。
* 地图矩阵 `matrix` 包含复杂的纵横交错迷宫路径、隐秘安全城镇大殿、以及死胡同深处的敌对山贼和宝箱。

### 5.2 室内外自适应倍率缩放
为了让室外宏大深邃、室内精致聚焦，系统设计了两套自适应视口：
1. **室外大地图 (Outdoor)**:
   * **视口尺寸**: `21 x 21` 瓦片（大幅扩展视野至原先的 5 倍）。
   * **单块大小**: `26px * 26px`，Canvas 物理显示尺寸 `546px * 546px`。
2. **室内场景 (Indoor)**:
   * **视口尺寸**: `11 x 11` 瓦片（聚焦近景，展现客栈大殿的精致感，放大 2 倍视野）。
   * **单块大小**: `44px * 44px`，Canvas 物理显示尺寸 `484px * 484px`。

### 5.3 精密相机边界裁剪 (Camera Boundary Calculation)
相机的投影位置 `(startX, startY)` 以玩家当前位置 `(px, py)` 为屏幕中心计算，同时使用严格取整和边界限位，彻底消除地图边缘渲染缺失、显示不全或越界溢出的问题：
```javascript
const startX = Math.max(0, Math.min(mapWidth - viewportWidth, Math.floor(px - viewportWidth / 2)));
const startY = Math.max(0, Math.min(mapHeight - viewportHeight, Math.floor(py - viewportHeight / 2)));
```

### 5.4 视野迷雾 (Fog of War)
* 玩家在每张地图上有一个独立的二维布尔数组 `explored`。
* 玩家初始周围有半径 3 格（曼哈顿距离）的视野。当玩家走动时，踩过、看过的格子均会被置为 `explored[x][y] = true`。
* **Canvas 遮罩层**：Canvas 绘图时，未探索的格子一律覆盖灰黑色迷雾遮罩，不绘制地貌和其中的 NPC / 传送门；仅已探索的格子展示全彩或纯正墨香地貌。

## 6. MUD 一轴式布局与无弹窗交互/战斗对话系统
### 6.1 单轴竖向水墨布局
彻底废除大地图侧边栏割裂设计，调整为垂直流向的**一轴水墨大视口布局**。大地图 Canvas 居中拉宽展示。

### 6.2 大地图正下方“多功能交互与战斗板”
原本散布在右侧的实体 Look 列表、NPC 对话框以及切磋比武的战斗数据显示，**100% 收纳整合在大地图 Canvas 正下方**。
* 当处于探索状态时，下方显示【当前格实体列表 LOOK 面板】与【卷轴式交互对话框】。
* **当开启战斗/比武时**，此面板无缝转化为 **【文字战斗日志与血条结算看板】**。玩家能够一边目睹 Canvas 上火柴人的对决动作，一边看到下方文字招式行云流水地更新。

### 6.3 彻底根除浏览器弹窗 (100% 零弹窗)
* 彻底废除 `alert()`, `confirm()` 等原生浏览器弹窗，保护国风武侠静谧的游戏体验。
* **水墨漂浮提示 (Ink Toast)**: 前端构建统一的 `#ink-toast` 容器。所有的挂机切换提示、登录失败、保存通知一律通过淡墨色、带淡入淡出的 Inline Toast 漂浮提醒。
* **内嵌错误拦截**: 创角出错、功法升级修为不够等情况，一律直接在下方对话框内容中以红色警示字提示（“*修为不足，无法参悟*”），避免产生弹窗阻断。

## 7. 战斗系统与 Canvas 火柴人绘制
### 7.1 战斗过程与跳过
* 战斗为系统自动运行的**半即时回合制**。
* 战斗开始时弹出战斗视窗，右下角提供 `【跳过战斗】` 按钮，点击直接在后台计算剩余回合并返回结算。

### 7.2 Canvas 火柴人动作绘制
* 战斗上方有一个专门的 Canvas 视口。
* **左侧火柴人（主角）**，**右侧火柴人（敌人）**。
* 双方有攻击进度条（根据敏捷决定）。
* **出招动画**: 进度条攒满的一方瞬间向另一方平移冲刺。
  * **武器大类动作表现**:
    * **拳脚**: 快速平移，挥动火柴人手臂拳击，打出气浪特效。
    * **刀/剑**: 手部延伸出一条黑线（刀/剑），冲刺并划过敌方身体，产生水墨刀光/剑气切痕。
    * **枪/棍**: 火柴人向前刺出长线条，并产生连击戳刺动画。
    * **鞭子**: 手部甩出一条波浪曲线，在敌方身上产生鞭击抽打与火花效果。
    * **内功爆发**: 角色立于原地，脚下浮现水墨八卦阵或波纹，敌方受到震退。
* **受击动画**: 被击中者向后倾斜抖动，爆出暴击、招架或闪避的浮空红白字，并附带击退。

## 8. 功法、装备与红颜系统
### 8.1 武学门类 (最高 10 层)
武学涵盖：**内功、轻功、武器（拳脚、刀、枪/棍、剑、鞭子）**。
* **层数上限**: 10 层。
* **消耗曲线**: 升级所需修为呈指数级递增：
  * `Cost(Level) = BaseCost * Math.pow(2.8, Level - 1)`
  * 1 级需要 100 修为，到 9 级、10 级将需要数十万乃至百万修为。
* **参悟机制**: 点击升级需要挂机消耗时间（受悟性缩短），且参悟期间角色不能进行其他挂机。

### 8.2 装备锻造
* 背包提供三个槽位：武器、衣服、配饰。
* 武器类型需与当前装备并使用的武器武学（刀/枪/剑/鞭）相匹配。
* 可在铁匠NPC格子上消耗**矿石资源+银两**进行强化与升星。

### 8.3 红颜伙伴
* 赠礼、聊天可培养与大地图隐居红颜/侠客的好感。
* 好感达到 100 结为伴侣/伙伴，可选择“协同出战”。

## 9. 代代相传——轮回衣钵系统 (Core Highlight)
### 9.1 领养与传位条件
* 当主角境界达到一定等级（如筑基/化境以上），且年龄达到 40 岁以后，可通过大地图上的特定剧情（如孤儿院或云游僧人）**领养一个孩子**。
* 玩家可以随时点击“【传承衣钵】”，将主角之位传给孩子。

### 9.2 轮回继承规则
1. **家传绝学**: 
   * 传位时，玩家可以从上一代主角生前掌握的所有武学中，**挑选一门武学（如：剑法）、一门轻功、一门内功**作为“家传绝学”传给孩子。
   * 孩子（下一世初始 15 岁）**出生即自动掌握**这三门绝学（1层）。
   * 并且后续升级这三门家传武功**完全不需要消耗参悟时间**，且升级所需要的修为消耗**永久降低为原来的 1/4**！
2. **父辈变成 NPC**:
   * 传承后，上一代主角（父辈）并不会消失，而是会**留存在大地图的祖宅（或门派洞府）中，变成一个可以交互的 NPC**。
   * 父辈 NPC 会继承其原有的【根骨】生命属性，并会随着真实时间流逝继续变老直至寿终正寝。
3. **传功与其它武学领悟**:
   * **传功功能（对家传绝学操作）**：在父辈老死之前，新主角（孩子）可以随时回到祖宅找到父辈 NPC 交互，点击 **【请教/传功】**。传功功能**仅能对选取的这三门家传武学进行操作**，传功将免除它们升级的参悟时间，并且修为消耗进一步减免！
   * **非家传武学升级加成**：对于孩子修炼的上一辈已学会的**非家传武学**，升级时的参悟时间以及修为消耗，将获得父辈遗留经验的比例化减免，具体计算公式为：
     $$\text{实际减免系数} = ((10 - \text{上一世该武学层数}) \times 0.05 + 0.5) \times 100\%$$
     *(注：上一世该武学层数越高，实际升级时消耗的时间与修为就越少，例如上一世该武学已达到 10 层，则孩子修炼该武功只需 50% 的时间和修为，极大降低了由于继承带来的重复肝度。)*
