# 2026-06-06 Godot 4.3 皇帝御案、天下舆图与动态朝臣面板设计规范

## 1. 设计目标

在现有的 C# 朝堂和地方割据后端逻辑基础之上，重构 Godot 4.3 前端，彻底改变此前“物理器物缺失、右侧朝臣死板、无法在图形界面平叛安抚”的空泛状态。
本规范将实装以下高保真交互部件：
- **御案四大金器常驻交互栏 (Emperor's Desk)**
- **“尚书台奏折卷宗”面板 (Affairs Panel)** - 漆木匣
- **“大汉十三州舆图情报”面板 (Intel Panel)** - 密札 (支持任命太守、派兵平叛、遣使安抚)
- **“宣政殿大朝会”面板 (Grand Court Panel)** - 玉玺 (触发 AI 群辩与三段式过渡)
- **右侧动态朝臣滚动列表 (Dynamic NPC Scroll List)** + 五维详情属性面板。

## 3. 御案物理器物实装方案

我们在 `CenterPanel` 的顶部，新增一个水平布局容器 `HBoxContainer` 作为 **Emperor's Desk (御案)**。
在 `MainScene.cs` 的 `_Ready` 中动态查找或生成以下四个按钮，并挂载真实交互回调：

```csharp
private HBoxContainer _deskContainer;
private Button _btnAffairsBox;      // 📜 雕龙漆木匣 (政务)
private Button _btnIntelToken;      // 🗺️ 漆木密札 (情报)
private Button _btnCourtSeal;       // 👑 天子玉玺 (朝会)
private Button _btnPleasureCenser;  // 💨 铜制博山炉 (娱乐)
```

当任意器物按钮点击时，使用 `WindowManager` 呼出对应的多层叠覆盖弹窗（所有弹窗统一由 C# 动态构建，在窗口退栈时自动 Hide）。

## 4. 四大核心交互面板设计

### 4.1 📜 “尚书台奏折卷宗”面板 (Affairs Panel)
- **触发**：点击 **雕龙漆木匣** 时，如果当前处于“宣政殿”，弹出此面板；如果处于“后宫”或“西园”，提示“陛下，奏折均存在宣政殿，请移步批阅”。
- **界面结构**：
  - 左半区：`ScrollContainer` + `VBoxContainer` 显示待批奏折列表（读取 `_gameState.ActiveEdicts`）。
  - 右半区：`RichTextLabel` 渲染当前选中奏折的叙事详情。
  - 右下区：`VBoxContainer` 动态加载该奏折的决策选项按钮。点击选项后，调用 `_gameEngine.ProcessPlayerTurnAsync()` 并触发国势更新，执行后关闭面板。
  - 底部：“暂缓批阅”按钮，退栈。

### 4.2 🗺️ “大汉舆图与军政情报”面板 (Intel Panel)
- **触发**：点击 **漆木密札** 时在任何场景均可弹出。
- **界面结构**：
  - **左半区 (十三州局势)**：`ScrollContainer` 纵向列出大汉 6 郡（司隶、冀州、并州、兖州、豫州、荆州）的缩影：
    - 每个郡显示一行：`【州郡名】 守军:xxxx | 民心:xx | 官:董卓 | 状态:安定/⚡叛乱(持续X个月)`。
    - 若该郡正处于 **黄巾之乱** 或 **民变/割据叛乱**，整行标红并闪烁 `⚡` 标识。
  - **右半区 (军政决策台)**：点击左侧任意郡后激活。
    - **A. 太守治理**：
      - 显示当前太守。提供 `【任命太守】` 或 `【召回朝廷】`。点击“任命太守”弹出当前闲置（未外任）的大臣列表。选择后执行 `_gameEngine.AssignGovernor()`。
    - **B. 兵戈平叛**（仅在目标郡 IsRebelling 为 true 时显示）：
      - 提示“大军平叛”。提供将领选择下拉框/按钮列表（仅显示在朝在京将领）。
      - 提供输入框决定调遣兵力数（上限为西园新军剩余总人数）。
      - 提示成功率公式预估值（将领战力、距离、兵力比折算）。
      - 点击 `【出征平叛】` 按钮，执行 `_gameEngine.SuppressRebellion()`，并在故事输出区展示战报。
    - **C. 遣使安抚**（仅在目标郡 IsRebelling 为 true 时显示）：
      - 提示“特使招抚”。选择在京大臣。
      - 提供 **四个复选框 (CheckBox)** 勾选安抚策略：
        - `[ ] 离间计 (需政治≥50)`
        - `[ ] 说服 (需魅力≥45)`
        - `[ ] 惩治官员 (需皇权≥35)`
        - `[ ] 赈灾放粮 (需拨国库 500-1500万钱)` —— 若勾选赈灾，显示滑块或数字框输入金额。
      - 显示预估招抚成功率。
      - 点击 `【下诏招安】`，执行 `_gameEngine.PacifyRebellion()`。

### 4.3 👑 “大朝会与AI群辩”面板 (Grand Court Panel)
- **触发**：点击 **天子玉玺** 时在宣政殿可弹出。
- **界面结构**：
  - 提示输入：“请陛下口授大朝会议题：”
  - 提供文本编辑框 `LineEdit` 键入如“赏赐何进”、“训诫张让”。
  - 点击 `【宣旨召见】`：
    - 自动触发朝仪换装、钟磬齐鸣等三阶段过渡文本（通过 6 秒缓动动画）。
    - 期间异步执行 `_gameEngine.ProcessPlayerTurnAsync(text)`，由 AI 调度师实时编织各派系口诛笔伐、迎合弹劾等精彩对话。
    - 渲染大屏直接输出百官群辩过程，扣人心弦。

## 5. 右侧动态 NPC 列表与五维详情

### 5.1 右侧列表重构
- 丢弃原有写死在 Tscn 里的四个大按钮。
- 新增 `ScrollContainer` + `VBoxContainer`（命名为 `NpcListContainer`）。
- 每次 `UpdateUI` 时，清空此容器，遍历 `_gameState.Npcs.Values` 中所有**处于活跃状态 (IsActive == true)** 的大臣。
- 动态生成一个按钮（Text 格式：“[派系] 太守/官职 姓名”）。
- 点击此按钮后，触发 `ShowMinisterDetails(npc.Id)` 面板。

### 5.2 大臣详情面板 (MinisterOverlayPanel) 升级
- 弹窗尺寸适当放大（如 400x320）。
- **五维显示**：除原本的名号、好感度、权势、贪腐和私蓄外，**新增一行横向网格/文字显式展示其五维数值**：
  - `武力: [Martial]  |  统帅: [Leadership]`
  - `政治: [Politics] |  魅力: [Charisma]`
  - `野心: [Ambition]` (野心值极高的官员派驻外任需极度慎重！)
- **地方官标识**：若其已任太守，加粗显示“【外任太守：冀州】”。
- **操作整合**：保留“抄家籍没（入国库/私库）”按钮。

## 6. 自我审查 (Self-Review)

1. **是否有 TBD 占位符？** 无，所有面板的数据结构与对应的后端 API 方法 (`AssignGovernor`, `SuppressRebellion`, `PacifyRebellion`) 均完全闭环对接。
2. **是否与 C# 逻辑引擎冲突？** 否，完全使用 Partial 机制和 existing APIs，直接驱动和同步修改 `GameState` 数据。
3. **性能与稳定性**：所有动态生成的 UI 在重新渲染时都先使用 `QueueFree()` 清理旧子节点，保证完全没有内存与垃圾对象积压。
