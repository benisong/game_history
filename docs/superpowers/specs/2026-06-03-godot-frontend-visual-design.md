# 2026-06-03 东汉末年汉灵帝：Godot 4.3 前端视觉美学与 UI 架构设计规范

## 1. 核心视觉美学 (Visual Aesthetic)
本项目的前端摈弃现代卡通或魔幻 UI，采用 **“汉代漆器 (Lacquered Wood) + 国风水墨 (Shanshui Ink) + 宫廷金石 (Patina Gold)”** 的沉浸式古典美学。营造一种深宫内苑、权谋重重、大厦将倾的压抑与厚重感。

### 1.1 核心配色 (Color Palette)
*   **玄黑大漆 (Lacquered Black, #0D0D0D)**：作为主场景案台与背景底色，代表水德大汉的威严与压抑。
*   **朱砂红 (Cinnabar Red, #A62424)**：用于朱批、重要警报、暗探密录的重点高亮，形成强烈的黑红视觉张力。
*   **青铜金 (Patina Gold, #B89047)**：用于 UI 边框、御玺、皇权等尊贵元素的镶边。
*   **生绢白/竹简青 (Washi Paper, #EAE5D9)**：用于奏章、竹简的正文底色，保障阅读清晰度而不刺眼。

## 2. 第一人称物态化大局布局 (Materialized Layout)

### 2.1 主界面：赤霄御案全景 (The Imperial Desk)
主场景 (`MainScene.tscn`) 采用第一人称俯视视角。
*   **上半部分 (30%)**：虚化的宣政殿大殿幽深全景。根据天子皇权和天下民心，大殿的光线会发生动态变明或变暗（动荡氛围）。
*   **下半部分 (70%)**：天子的黑漆龙案。案台上放置四个具备 2D 骨骼/粒子交互动效的**物理物件**，分别对应四大核心入口：

| 物理物件 (UI Button / TextureRect) | 隐喻入口 | 交互动效预想 |
| :--- | :--- | :--- |
| **雕龙漆木匣** | **【政务处理】** | 点击时，木匣盖子滑开，传来木枢转动声。 |
| **漆木密札** | **【黄门暗探】** | 点击时，伴随竹简碰撞与微弱锁链声。 |
| **天子和氏璧玉玺** | **【召集朝会】** | 点击时，玉玺重重砸下（屏幕震动），开启朝仪。 |
| **铜制博山炉** | **【饮酒娱乐】** | 点击时，炉顶喷出紫色缭绕烟雾粒子，覆盖全屏。 |

## 3. 辅界面展开与交互隐喻 (Sub-Panels)

辅界面不再是硬生生的 `PopupPanel`，而是融入游戏世界观的器物展开：

### 3.1 【漆匣奏章】（政务批复）
*   **展开方式**：木匣滑开后，奏章以一卷生绢的形式在屏幕中央向两侧徐徐拉开 (`Tween` 缓动动画)。
*   **交互隐喻**：界面下方提供一个**“朱砂笔托”**区域。点击 3 个决策选项，伴随毛笔在纸上书写的沙沙声，选项化为朱红字迹落印在奏章尾部，完成御批。

### 3.2 【黄门暗探】（情报系统）
*   **展开方式**：屏幕变暗，一捆血色与暗墨交织的粗糙竹简铺开。
*   **视觉呈现**：
    *   竹简左侧展示大汉十三州地图缩略图（血斑代表黄巾、天灾预警）。
    *   竹简右侧挂着百官的**“木制身份牌”**。
    *   **藏锋于词**：鼠标悬停（Hover）在官员木牌上时，用行楷字体浮现其性格特征标签（如 `[经天纬地]`、`[贪得无厌]`）。

### 3.3 【大朝会】（朝堂博弈）
*   **展开方式**：天子盖下玉玺，全屏被纯黑的 **大朝仪过渡遮罩（换装 -> 宫门开 -> 鸣磬）** 覆盖。
*   **视觉呈现**：
    *   遮罩褪去后，视角抬起直视宣政殿。两侧展示百官剪影立绘。
    *   打头阵的奏折悬浮在中间。左右两侧以古风对话气泡（犹如唇枪舌剑）展示张让、何进等 AI 调度生成的多角色阵营弹劾与辩驳。

### 3.4 【饮酒娱乐】（后宫/西园）
*   **展开方式**：博山炉紫烟散去后，进入充满暗红纱帐与暖色宫灯的内苑辅界面。
*   **视觉呈现**：慵懒、奢华。选项雕刻在青铜酒樽之上，伴随靡靡的古筝或编钟背景音。

## 4. Godot 前端节点架构预想

为了实现这种物态化和极具层级感的 UI，推荐采用以下节点架构 (`MainScene.tscn`)：

```text
Node2D (MainScene)
├── TextureRect (Background_Hall)             # 上半部分：宣政殿虚化背景
├── TextureRect (Imperial_Desk)               # 下半部分：赤霄御案
│   ├── TextureButton (Btn_Affairs_Box)       # 漆木匣
│   ├── TextureButton (Btn_Intel_Token)       # 密札
│   ├── TextureButton (Btn_Court_Seal)        # 玉玺
│   └── TextureButton (Btn_Pleasure_Censer)   # 博山炉
│
├── CanvasLayer (UI_Layer)                    # 独立于世界层之上的 UI 渲染层
│   ├── Control (Panel_Affairs)               # 奏折展开界面 (缓动隐藏/显示)
│   ├── Control (Panel_Intel)                 # 黄门竹简界面
│   ├── Control (Panel_Pleasure)              # 纱帐内苑界面
│   └── ColorRect (Transition_Mask)           # 全屏黑底遮罩 (用于大朝仪 3 阶段转场)
```

## 5. 自我审查 (Self-Review)
*   **美学一致性**：四大模块皆依托于汉代器物（木匣、竹简、玉玺、青铜炉），消除了软件 UI 感。
*   **性能考量**：辅界面的开启与闭合通过 Godot 的 `Tween` 修改 `modulate.a` (透明度) 和 `position` (位置) 实现，资源开销极小，保证 60FPS 顺滑。
